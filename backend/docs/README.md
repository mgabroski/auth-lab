# Backend Docs Map

This folder contains backend-specific documentation.

## Read in this order for backend work

1. `../AGENTS.md`
2. `engineering-rules.md`
3. `module-skeleton.md`
4. relevant files under `api/`

## What lives here

- `engineering-rules.md` -> backend implementation law
- `module-skeleton.md` -> standard backend module shape
- `api/` -> backend-owned domain API contracts
- `modules/` -> opt-in module-local docs only for genuinely complex modules
- `adr/` -> backend architectural decision records and references when needed

## Important rule

Do not treat this file as a second router or a second architecture doc.
It is only a folder map.
